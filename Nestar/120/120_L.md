***************************************************************************
**standard mode: **
🔹 Standard mode
  Bu oddiy loyiha tuzilmasi.
    👉 Har bir loyiha alohida bo‘ladi
    👉 Bitta repo = bitta project
Misol:
  my-app/
   ├── frontend/
   ├── backend/

yoki ba’zida faqat:
my-app/
 ├── src/

📌 Qachon ishlatiladi:
  Kichik yoki o‘rta loyihalarda
  Bitta app bo‘lsa
  Oddiy ishlash kerak bo‘lsa
***************************************************************************

** monorepo mode:  **
🔹 Monorepo mode
  Bu katta loyiha uchun.
    👉 Bitta repo ichida bir nechta projectlar bo‘ladi
    👉 Hammasi bitta joyda boshqariladi
Misol:
  my-monorepo/
   ├── apps/
   │    ├── web/
   │    ├── mobile/
   ├── packages/
   │    ├── ui/
   │    ├── utils/
📌 Qachon ishlatiladi:
Katta systemalarda
Frontend + Backend + Mobile birga bo‘lsa
Kodni reuse qilish kerak bo‘lsa
***************************************************************************
 "build": "nest build",                                       // Loyihani build qiladi (TypeScript → JavaScript, dist papkaga chiqaradi)
"format": "prettier --write "apps/**/*.ts" "libs/**/*.ts"",   // Kodni avtomatik tartiblaydi (format qiladi, chiroyli qiladi)
"start": "nest start",                                        // Oddiy rejimda appni ishga tushiradi
"start:dev": "nest start --watch",                            // Development rejim (kod o‘zgarsa avtomatik qayta ishga tushadi)
"start:dev:batch": "nest start nestar-batch --watch",         // Batch (alohida service/app) ni dev rejimda ishga tushiradi
"start:debug": "nest start --debug --watch",                  // Debug rejim (breakpoint qo‘yib tekshirish uchun)
"start:prod": "node dist/apps/nestar-api/main",               // Production build bo‘lgan API ni ishga tushiradi
"start:prod:barch": "node dist/apps/nestar-batch/main"        // Production batch service ni ishga tushiradi

***************************************************************************

** @nestj/config ** .env fayllar uchun install qilib olamiz
@nestjs/config — bu environment (env) o‘zgaruvchilarni boshqarish uchun modul.
👉 Ya’ni .env fayldagi ma’lumotlarni ishlatish uchun kerak

***************************************************************************

  👉 NestJS’da har qanday modulni ishlatish uchun uni import qilish kerak

    imports: [ConfigModule.forRoot()],    //forRoot() → env ni yuklaydi va aktiv qiladi
    
        .env faylni o‘qiydi
        process.env ga yuklaydi
        butun app bo‘ylab ishlatishga imkon beradi
